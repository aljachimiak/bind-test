This is a Reveal.js presentation

Reveal.js is a javascript library that makes it easy to create attractive browser based presentations in with html, css, and javascript.

Feel free to click around!

Main Goal of this project:

Create an in-browser method to edit and simultaneously preview a Reveal.js presentation

This entails generating html in the browser and handing off the markup to Reveal.js all within the same user interface.

Problems:

  • Contain Reveal.js
    As you can see, Reveal takes up the entirety of the space it is given. To preview in the same interface as we edit, Reveal will be contained in an iframe.
  • Binding
    As we create new text and markup, it will need to be passed into the iframe dynamically for Reveal to parse. This is achieved with Facebooks's React.js library.

More Problems:

  • Generating markup
    Browsers are made to parse html. So writing html can get difficult. CKEditor is used as a solution to have an 'WYSIWYG' way to produce markup.
  • Custom Plugin
    Reveal uses 
    <section>
    elements to create different slides. I made a custom CKEditor plugin that inserts a 
    <hr class="newSlide" />
    to split strings and create new presentation pages.

Other neat stuff...

  • Using React.js to insert script tags into the head of an iFrame was difficult and worth noting.
  • using viewport height and width units to responsively determine the size of the Reveal iFrame
  • Use Javascript's .toString() reflection on relevant functions to display code on each page. It made for some odd js, but was meant as a way to reduce the need to open up an editor to see code.
  • Using the css counter property in several different ways (code display, enumerating sections)
